Some things you may want to know...

  • Templates (.jinja for Jinja2 through the Coffin library; .html for Django Templates) live underneath templates/ (that is {{ settings.TEMPLATE_DIRS.0 }}) are mapped directly to URLs.
    That is, if you go to /test/, either templates/test.html or templates/test.jinja will be rendered, depending on which one can be found.
  • Static files live underneath static/ (that is {{ settings.STATICFILES_DIRS.0 }}) and are mapped to {{ STATIC_URL }}.
    • The Yeti Bootstrap 3 theme is available as bootstrap3.yeti.css directly in the static file root.
    • (And so is that lovingly hand-traced Django Pony, as pwny.png.)
  • A number of pseudo-model objects are made available for templates. You can see them in action over at /models/.
    Each pseudo-model has at least {{ object.keys|length }} different attributes – see the Models page for a list.
    Reasonable care is taken to try and ensure the objects do not change between page loads (though each URL gets a different set of objects). This, however, depends on how one iterates over their attributes.
    • object is a single pseudo-object.
    • ten_objects is a list of 10 pseudo-objects.
    • fifty_objects is a list of 50 pseudo-objects.
    • paginator is a paginator of 5000 pseudo-objects, 30 per page.
